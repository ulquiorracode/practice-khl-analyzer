# Журнал изменений (Changelog)

Все заметные изменения в этом проекте будут отражены в данном файле.

Формат ведения журнала основан на принципах [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
и проект придерживается [семантического версионирования](https://semver.org/lang/ru/v2.0.0.html).

## [0.4.0] — 2026-06-29

### Добавлено

- Новый аргумент командной строки `--standings-mode` с тремя режимами отображения таблиц:
  - `both` — совместный вывод итоговой таблицы и таблиц конференций (по умолчанию).
  - `championship` — вывод только общей таблицы регулярного чемпионата.
  - `conferences` — вывод только таблиц конференций.
- Новые разделы в `README.md` с подробным разбором, аналогиями и примерами для векторных методов Pandas (`.melt()`, `.apply()`, `.to_dict()`, `.map()`, `.fillna()`, `.to_numeric()`, `.copy()`).
- Документирование новых сценариев запуска CLI в файле `README.md` [3].

### Изменено (Рефакторинг под требования Pandas-ориентированности)

- **Исключение ветвлений в аналитическом движке:** В методе `get_conference_tables` класса `KHLSeasonAnalyzer` полностью удалены тернарные операторы и условные конструкции `if`. Сопоставление клубов с их дивизионами теперь реализовано в виде единой векторной цепочки Pandas с использованием `pd.DataFrame.melt()`, `.explode()` и математического сдвига индексов строк.
- **Исключение Python-циклов в аналитическом движке:** Генератор словаря (dict comprehension) для получения таблиц конференций заменен на полностью векторную конструкцию Pandas: `pd.Series(index).apply(func).to_dict()`.
- **Исключение ветвлений в CLI-модуле:** Логика вывода таблиц в функции `run_standings()` (`main.py`) избавлена от условных операторов `if/else` и переписана на основе декларативного словаря-диспетчера (`mode_dispatcher`) [2].
- **Динамический заголовок конференций:** Реализован вывод актуального названия конференции («Запад»/«Восток») в заголовке таблицы [2]. Для сохранения Pandas-стилистики вывод реализован через `.apply()` по серии ключей словаря таблиц конференций.

### Безопасность и стабильность

- Новая векторная логика на базе `.melt()` спроектирована с учетом обработки граничных сценариев: при отсутствии дивизионов в конфигурации сезона (например, для сезонов до 2017 года) методы отрабатывают без ошибок за счет безопасных цепочек преобразования пустых структур и возвращают пустой объект `Series`, который затем корректно заполняется значениями `-1` через `.fillna()`.

## [0.3.0] — 2026-06-24

### Добавлено

- Новый класс `HtmlPrinter` в пакете `printers/` для генерации HTML-кода турнирных таблиц на основе встроенного векторного метода `pandas.DataFrame.to_html()`.
- Добавлен параметр командной строки `--output` для записи выходных данных напрямую в файл из среды Python, минуя интерпретатор командного процессора.
- В интерфейс базового класса `BasePrinter` и во все его реализации добавлен параметр `stream` (поток вывода) для поддержки записи таблиц в любые файлоподобные объекты.
- Соответвующие примеры использования новых функций добавлены в `README.md`.

### Изменено

- Двухпозиционный флаг `--pretty` удален и заменен на универсальный строковый параметр `--format` с опциями: `text` (простой текст), `pretty` (красивая сетка) и `html` (HTML-таблица).
- Код всех принтеров переведен со стандартной функции `print()` на явную запись в переданный поток через метод `stream.write()`.
- Выбор целевого потока вывода (системный файл или стандартный вывод `sys.stdout`) реализован через функциональный словарь лямбда-выражений без использования операторов ветвления `if/else`.

### Исправлено

- Исправлена критическая ошибка `UnicodeEncodeError` в ОС Windows при перенаправлении потоков (использовании операторов `>` или `|`) за счет принудительной конфигурации стандартного вывода `sys.stdout` на кодировку `UTF-8`.
- Устранены визуальные артефакты искажения кириллицы (проблема двойного перекодирования PowerShell) при генерации HTML-отчетов с помощью внедрения прямого файлового вывода через флаг `--output` в кодировке `UTF-8`.

## [0.2.0] — 2026-06-24

### Добавлено

- Создана модульная архитектура проекта с разделением на подпакеты: `models/` для бизнес-сущностей и `printers/` для механизмов отображения.
- Реализована модель данных сырого матча КХЛ (`MatchSchema`) для исключения «магических строк» (magic strings) и жестких привязок к именам столбцов CSV.
- Реализован паттерн «Стратегия» (Strategy Pattern) для вывода таблиц в консоль на основе абстрактного класса `BasePrinter`.
- Добавлен класс `PrettyConsolePrinter` с интеграцией внешней библиотеки `tabulate` для отрисовки структурированных Unicode-сеток.
- Добавлен логический ключ командной строки `--pretty` для переключения между простым и структурированным режимами отображения таблиц.
- Добавлена конфигурация проекта по современным стандартам PEP 621/517 через файл `pyproject.toml`.
- Внедрена строгая аннотация типов (`Type Hints` из модуля `typing`) для всех интерфейсов и методов проекта.

### Изменено

- Выполнен глубокий рефакторинг монолитного скрипта `khl_standings.py`. Код разделен на независимые модули: `models/`, `printers/`, `analyzer.py` и `main.py`.
- Логика ветвления в диспетчере задач CLI и при выборе принтера переписана на использование структуры словарей (Dictionary Dispatch) вместо операторов `if/else`.
- Логика связывания конференций и дивизионов переведена на векторный метод `pandas.DataFrame.explode()`, что позволило полностью уйти от циклов `for`.
- Обновлена сопроводительная документация: во все модули добавлены подробные русскоязычные комментарии, описывающие специфику применения векторных операций Pandas и NumPy.

### Исправлено

- Исправлена ошибка `AttributeError` при вызове индивидуальных задач, вызванная отсутствием методов визуализации графиков внутри класса `KHLSeasonAnalyzer` после его вынесения в отдельный модуль.
- Восстановлены зависимости библиотеки `matplotlib` внутри расчетного сервиса `analyzer.py`.
