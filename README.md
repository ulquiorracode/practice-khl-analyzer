## Запуск программы

Чтобы запустить программу, необходимо передать в CLI обязательные параметры:

```bash
python main.py <path_to_csv>
```

Где:

- `<path_to_csv>` - путь до файла с данными

Дополнительные параметры:

- `--task` - задание, которое необходимо выполнить
- `--team` - команда, для которой необходимо выполнить задание
- `--format` - формат вывода (text, pretty, html)
- `--output` - вывод в файл (по умолчанию в консоль)
- `--standings-mode` - режим вывода таблиц для задачи "standings":
  - both - выводить общую таблицу и таблицы конференций вместе (по умолчанию)
  - championship - выводить только итоговую таблицу чемпионата
  - conferences - выводить только таблицы конференций

### ЧАСТЬ 1. ПОДРОБНЫЙ ГАЙД ПО ФУНКЦИЯМ БИБЛИОТЕК

#### 1. Векторные операции в Pandas (Сравнения и замена данных)

В Pandas операции выполняются сразу над всем столбцом без использования циклов `for`. Это называется **векторизацией**.

##### `.mask(cond, other)` — Условная замена («Малярный скотч»)

- **Детальное описание:** Метод проверяет каждую ячейку столбца на соответствие условию `cond`. Если условие выполняется (`True`), старое значение заменяется на `other`. Если условие не выполняется (`False`), значение остаётся нетронутым.
- **Аналогия:** Представьте малярный скотч. Вы наклеиваете трафарет (условие) на стену. Те места, которые попали в вырезы трафарета, вы закрашиваете новой краской (`other`), а закрытые скотчем места остаются прежнего цвета.
- **Пример:**

    ```python
    import pandas as pd
    s = pd.Series([1, 10, 3, 12])
    # Заменим все числа, которые больше 5, на число 99
    result = s.mask(s > 5, 99)
    # Результат: [1, 99, 3, 99]
    ```

##### `.gt()`, `.lt()`, `.eq()` — Быстрые сравнения

- **Детальное описание:** Эти методы расшифровываются как **G**reater **T**han (больше), **L**ess **T**han (меньше) и **E**qual (равно). Они сравнивают элементы столбца с числом или другим столбцом и возвращают столбец, состоящий только из `True` и `False`.
- **Аналогия:** Пропускной контроль на заводе. Металлоискатель проверяет каждого человека индивидуально и выдает сигнал: «Да» (металл есть) или «Нет» (металла нет). На выходе мы получаем цепочку сигналов для всей очереди.
- **Пример:**

    ```python
    s = pd.Series([2, 5, 8])
    print(s.gt(5)) 
    # Вывод:
    # 0    False (2 не больше 5)
    # 1    False (5 не больше 5)
    # 2     True (8 больше 5)
    ```

##### `.map(arg)` — Замена значений по словарю («Справочник шифров»)

- **Детальное описание:** Преобразует каждое значение в столбце на основе переданного словаря или другой серии `arg`. Находит старое значение среди ключей словаря и подставляет вместо него соответствующее значение.
- **Аналогия:** Работа шифровальщика. У вас есть список странных букв: `["В", "ВО", "П"]`. И есть бумажный справочник: `{"В": 3, "ВО": 2, "П": 0}`. Метод `.map()` берет каждую букву по очереди, находит ее в справочнике и записывает количество очков прямо на ее место.
- **Пример:**

    ```python
    s = pd.Series(["В", "ВО", "П"])
    points = {"В": 3, "ВО": 2, "П": 0}
    print(s.map(points))
    # Вывод: [3, 2, 0]
    ```

##### `.fillna(value)` — Заполнение пустот («Шпатлевка трещин»)

- **Детальное описание:** Находит все пропущенные или неопределенные значения (`NaN`) в столбце и заменяет их на указанное фиксированное значение `value`.
- **Аналогия:** Вы построили стену (таблицу), но в ней местами остались дыры и пустоты от выпавших кирпичей (`NaN`). Чтобы стена не рухнула и выглядела аккуратно, вы берете шпатлевку (`value`, например, число `-1`) и аккуратно затираете все пустоты.
- **Пример:**

    ```python
    import numpy as np
    s = pd.Series([12, np.nan, 45])
    print(s.fillna(-1))
    # Вывод: [12.0, -1.0, 45.0]
    ```

##### `pd.to_numeric(arg, errors="coerce")` — Безопасное приведение типов («Таможенный досмотр»)

- **Детальное описание:** Пытается преобразовать элементы столбца в числа. Если попадается бракованное значение (буквы, мусор), параметр `errors="coerce"` принудительно заменяет его на `NaN`, предотвращая аварийную остановку программы.
- **Аналогия:** На склад приходят коробки с надписями. Большинство из них содержат числа вроде `"5"` или `"3"`, но в некоторых лежит случайный текст. Метод `to_numeric` распаковывает их и переводит строки в математические числа. Опция `coerce` — это строгий таможенник, который не останавливает конвейер при виде бракованной коробки, а просто наклеивает на нее ярлык «пусто» (`NaN`) и пропускает дальше.
- **Пример:**

    ```python
    s = pd.Series(["10", "ошибка", "20"])
    print(pd.to_numeric(s, errors="coerce"))
    # Вывод: [10.0, NaN, 20.0]
    ```

##### `.copy()` — Создание независимой копии («Ксерокопия документов»)

- **Детальное описание:** Создает физически независимую копию объекта в оперативной памяти компьютера. Позволяет изменять скопированные данные без риска случайно испортить оригинал.
- **Аналогия:** Вы пришли в архив и взяли важную оригинальную карту. Чтобы сделать на ней пометки фломастером, вы не пишете на оригинале. Вы сначала делаете ксерокопию (`.copy()`), уносите её за свой рабочий стол и смело наносите изменения. Оригинал в архиве остается в полной безопасности. Также это предотвращает предупреждение `SettingWithCopyWarning`.
- **Пример:**

    ```python
    original_df = pd.DataFrame({"Данные": [1, 2, 3]})
    new_df = original_df.copy()
    new_df["Данные"] = new_df["Данные"] * 10  # original_df не изменится!
    ```

##### `.apply(func)` — Универсальная обработка («Индивидуальный конвейер»)

- **Детальное описание:** Применяет вашу пользовательскую функцию `func` к каждому элементу столбца (или строки) по очереди, возвращая новый результат.
- **Аналогия:** Представьте конвейерную ленту, по которой едут детали. Вы ставите у ленты мастера (функцию `func`), который берет каждую деталь, производит над ней сложную ручную операцию и кладет обратно на ленту. Метод `.apply()` позволяет запустить любой нестандартный процесс обработки без написания циклов.
- **Пример:**

    ```python
    s = pd.Series(["Запад", "Восток"])
    # Применяем функцию, которая делает текст заглавным
    print(s.apply(lambda x: x.upper()))
    # Вывод: ["ЗАПАД", "ВОСТОК"]
    ```

#### 2. Группировка и пересчет данных

##### `.groupby(by)` — Разделение на группы («Сортировка документов»)

- **Детальное описание:** Группирует строки таблицы по уникальным значениям в выбранном столбце `by`. Сама по себе эта функция не производит вычислений, она лишь подготавливает данные к расчету, раскладывая их по "виртуальным папкам".
- **Аналогия:** Представьте кучу чеков из супермаркета. Вы раскладываете их по стопкам: «Продукты», «Бытовая химия», «Одежда». Разложив их, вы еще не знаете общих сумм расходов, вы просто подготовили стопки.
- **Пример:** См. ниже в связке с `.agg()`.

##### `.agg(функция)` — Вычисление показателей внутри групп

- **Детальное описание:** Применяет математическую функцию (сумму, среднее, количество) к каждой группе, созданной с помощью `.groupby()`.
- **Аналогия:** Вы берете стопку чеков «Продукты» и складываете на калькуляторе все суммы в ней. Затем берете стопку «Одежда» и делаете то же самое. На выходе у вас получается один итоговый лист с суммами по каждой категории.
- **Пример:**

    ```python
    df = pd.DataFrame({
        'Категория': ['Продукты', 'Одежда', 'Продукты'],
        'Расход': [500, 2000, 300]
    })
    # Группируем по категории и считаем сумму расходов в каждой
    summary = df.groupby('Категория').agg(Всего_потрачено=('Расход', 'sum'))
    # Результат:
    # Категория | Всего_потрачено
    # Одежда    | 2000
    # Продукты  | 800
    ```

##### `.pivot_table()` — Сводная таблица («Двумерный отчет»)

- **Детальное описание:** Строит перекрестную таблицу, группируя данные одновременно по строкам (`index`) и по столбцам (`columns`). На пересечении строк и столбцов вычисляется агрегированное значение.
- **Аналогия:** Школьный классный журнал. По вертикали идут имена учеников, по горизонтали — даты уроков, а на пересечении стоят оценки. Это позволяет увидеть успеваемость конкретного ученика в конкретный день.
- **Пример:**

    ```python
    df = pd.DataFrame({
        'Команда': ['Спартак', 'Спартак', 'ЦСКА', 'ЦСКА'],
        'Исход': ['В', 'П', 'В', 'В'],
        'Матчи': [1, 1, 1, 1]
    })
    # Считаем количество побед ("В") и поражений ("П") для каждой команды
    pivot = df.pivot_table(index='Команда', columns='Исход', values='Матчи', aggfunc='count', fill_value=0)
    # Результат:
    # Команда | В | П
    # Спартак | 1 | 1
    # ЦСКА    | 2 | 0   (благодаря fill_value=0 пустая ячейка заменена на ноль)
    ```

##### `.cumsum()` — Нарастающий итог («Копилка»)

- **Детальное описание:** Считает сумму элементов столбца сверху вниз. Каждая новая ячейка хранит сумму текущего элемента и всех элементов, которые стояли выше него.
- **Аналогия:** Вы каждый день бросаете монеты в прозрачную копилку. Нас интересует не то, сколько вы бросили сегодня, а сколько денег лежит в копилке на сегодняшний день. В первый день: 5р (в копилке 5). Во второй день: +10р (в копилке 15). В третий день: +2р (в копилке 17).
- **Пример:**

    ```python
    points = pd.Series([3, 0, 2, 1])
    cum_points = points.cumsum()
    # Результат: [3, 3, 5, 6]  (накопление очков по ходу сезона)
    ```

#### 3. Изменение структуры таблиц

##### `.explode(column)` — Раскрытие списков («Коробка с апельсинами»)

- **Детальное описание:** Если в ячейке таблицы лежит список элементов, `.explode()` превращает этот список в отдельные строки, дублируя при этом всю остальную информацию из исходной строки.
- **Аналогия:** Представьте посылку. На ней написан адрес доставки, а внутри лежат три одинаковые футболки разного цвета (красная, синяя, зеленая). Метод `explode` — это когда мы берем три отдельные коробки, в каждую кладем по одной футболке и на каждую наклеиваем точно такой же адрес доставки, какой был на большой коробке.
- **Пример:**

    ```python
    df = pd.DataFrame({
        'Дивизион': ['Боброва'],
        'Клубы': [['СКА', 'Спартак']]  # Внутри ячейки лежит список
    })
    # Раскрываем столбец со списком клубов
    exploded_df = df.explode('Клубы')
    # Результат:
    # Дивизион | Клубы
    # Боброва  | СКА
    # Боброва  | Спартак
    ```

##### `.melt(value_name)` — Преобразование таблицы из широкой в длинную («Расплющивание коробки»)

- **Детальное описание:** Поворачивает таблицу так, что названия столбцов становятся значениями в одной новой колонке (`variable`), а данные внутри столбцов переносятся в другую колонку (`value`).
- **Аналогия:** Представьте школьное расписание, где по горизонтали написаны дни недели (Понедельник, Вторник), а под ними — списки предметов. Метод `.melt()` превращает эту плоскую сетку в один длинный узкий свиток: «Понедельник — Математика», «Понедельник — Физика», «Вторник — Химия».
- **Пример:**

    ```python
    df = pd.DataFrame({
        "Запад": [["СКА", "Йокерит"]],
        "Восток": [["Ак Барс", "Трактор"]]
    })
    # Превращаем колонки "Запад" и "Восток" в строки
    df_melted = df.melt(value_name="Клубы", ignore_index=False)
    # Результат:
    # variable | Клубы
    # Запад    | ['СКА', 'Йокерит']
    # Восток   | ['Ак Барс', 'Трактор']
    ```

##### `.to_dict()` — Экспорт данных в словарь («Переводчик на язык Python»)

- **Детальное описание:** Конвертирует Pandas Series или DataFrame в стандартный Python-словарь (`dict`), где индексами становятся ключи, а ячейками — значения.
- **Аналогия:** Вы составили список покупок на бланке Pandas, где слева напечатан порядковый номер или название (индекс), а справа — товар. Но ваш кассовый аппарат не знает, что такое Pandas, и понимает только встроенный формат Python. Метод `.to_dict()` переводит этот список в стандартный формат фигурных скобок `{ключ: значение}`.
- **Пример:**

    ```python
    s = pd.Series([10, 20], index=["ЦСКА", "СКА"])
    print(s.to_dict())
    # Вывод: {"ЦСКА": 10, "СКА": 20}
    ```

#### 4. Визуализация в Matplotlib

При вызове `fig, ax = plt.subplots()` создается два объекта:

1. **`fig` (Figure)** — это само белое бумажное полотно (окно), на котором мы рисуем.
2. **`ax` (Axes)** — это конкретная координатная сетка (оси X и Y) внутри этого полотна, на которой строится график.

##### `ax.plot()` — Линейный график

- **Детальное описание:** Соединяет точки с координатами $(x, y)$ непрерывной линией.
- **Аналогия:** Детская игра «соедини точки по номерам», чтобы получился рисунок животного. Каждая точка — это конкретная дата и количество набранных к этому дню очков.

##### `ax.hist()` — Гистограмма (Столбчатая диаграмма распределения)

- **Детальное описание:** Считает частоту попадания чисел в определенные числовые интервалы и рисует их в виде вертикальных столбцов. Высота столбца показывает, как много элементов попало в этот интервал.
- **Аналогия:** Сортировка людей по росту. Мы делаем три интервала: "низкие" (до 160 см), "средние" (160–180 см) и "высокие" (выше 180 см). Затем строим столбцы: высота первого столбца — сколько человек низкого роста, второго — сколько среднего и так далее.

### ЧАСТЬ 2. ГЛОССАРИЙ ТЕРМИНОВ ДЛЯ СДАЧИ ПРОЕКТА

Этот раздел поможет вам правильно отвечать на теоретические вопросы преподавателя во время защиты.

| Термин | Простое объяснение | Пример из кода проекта |

## Безопасность при конвертации в int

Анализ кода на предмет ненужных конвертаций в `int` и потенциальных проблем с невалидными данными.

### Анализ конвертаций в `int` и безопасности

**1. `_parse_score` метод:**

- **Конвертация:** `pd.to_numeric(..., errors="coerce").fillna(0).astype(int)`
  - **Назначение:** Парсит строковые значения счета (например, "3:2") в целые числа.
  - **Безопасность:** **Очень безопасно**.
    - `errors="coerce"`: Если парсинг не удается (например, вместо числа приходит текст), вместо ошибки возвращается `NaN`.
    - `fillna(0)`: Заменяет все `NaN` (которые возникли из-за `coerce` или были пустыми в исходном CSV) на `0`.
    - `astype(int)`: Преобразует получившиеся числа (или нули) в целочисленный тип.
  - **Ненужность:** Конвертация в `int` здесь **необходима** для дальнейших арифметических операций (сложение голов, сравнение). Без этого нельзя было бы посчитать сумму голов или определить победителя.

**2. `_process_match_outcomes` метод:**

- **Конвертация 1:** `home_code.map(POINTS_MAP).astype(int)` и `away_code.map(POINTS_MAP).astype(int)`

  - **Безопасность:** **Безопасно**. `POINTS_MAP` — это `pd.Series`, который содержит все возможные исходы, и `map` работает надежно. Даже если в `home_code` или `away_code` появится неожиданный символ, `map` вернет

- **Конвертация 1:** `home_code.map(POINTS_MAP).astype(int)` и `away_code.map(POINTS_MAP).astype(int)`
  - **Назначение:** Преобразует буквенные коды исходов ("В", "ВО", "П" и т.д.) в соответствующие им очки (3, 2, 1, 0).
  - **Безопасность:** **Безопасно**. `POINTS_MAP` — это `pd.Series`, который содержит все возможные исходы, и `map` работает надежно. Даже если в `home_code` или `away_code` появится неожиданный символ, `map` вернет `NaN`, который затем будет преобразован в `0` (см. пункт 3).
  - **Ненужность:** Конвертация в `int` здесь **необходима**, так как очки — это числовые значения, которые потом суммируются (`Очки_матч: "sum"`).

- **Конвертация 2:** `reg_home_win.astype(int)` и т.д. (для `ot_home_win`, `so_away_win`)
  - **Назначение:** Булевы серии (`True`/`False`) преобразуются в `1`/`0`.
  - **Безопасность:** **Полностью безопасно**. `astype(int)` на булевых значениях всегда работает предсказуемо.
  - **Ненужность:** Конвертация **не является строго необходимой** для логики, но она используется для удобства при добавлении гола за победный буллит:
    - `h_goals: pd.Series = total_h + so_home_win.astype(int)`
    - Если `so_home_win` (победа хозяев в буллитах) — `True`, то `astype(int)` превращает её в `1`, и к `total_h` добавляется 1 гол (победный буллит). Если `False` — добавляется `0`.
    - Без этой конвертации пришлось бы использовать более сложные конструкции `np.where` или `pd.Series.mask`. Так сделано лаконичнее.

- **Конвертация 3:** `pd.to_numeric(..., errors="coerce").fillna(0).astype(int)` внутри `_process_match_outcomes` (по аналогии с `_parse_score`, но в данном случае эти парсинги уже были выполнены в `_parse_score` и их результаты переданы сюда).
  - **Назначение:** Как в `_parse_score`, парсит строки с голами в числа.
  - **Безопасность:** **Очень безопасно**, как описано в пункте 1.
  - **Ненужность:** **Необходима** для подсчета голов и разницы шайб.

**3. `_build_global_standings` метод:**

- **Конвертация 1:** `.reindex(columns=["В", "ВО", "ВБ", "ПБ", "ПО", "П"], fill_value=0).astype(int)`
  - **Назначение:** Убеждается, что все колонки исходов присутствуют (даже если по ним нет данных), и заполняет отсутствующие значения нулями. Затем преобразует их в `int`.
  - **Безопасность:** **Безопасно**. `fill_value=0` гарантирует, что отсутствующие значения станут нулями. `astype(int)` корректно работает с нулями.
  - **Ненужность:** **Необходима** для последующего суммирования (`.sum(axis=1)`), чтобы посчитать общее количество игр (`И`).

- **Конвертация 2:** `standings["И"] = standings[["В", "ВО", "ВБ", "ПБ", "ПО", "П"]].sum(axis=1)`
  - **Назначение:** Считает общее количество сыгранных игр.
  - **Безопасность:** **Безопасно**, так как все колонки (`В`, `ВО` и т.д.) уже были приведены к `int` (или `float` с нулями) на предыдущем шаге.
  - **Ненужность:** **Необходима** для отображения в турнирной таблице.

- **Конвертация 3:** `standings["Разница"] = standings["Забито"] - standings["Пропущено"]`
  - **Назначение:** Вычисляет разницу шайб.
  - **Безопасность:** **Безопасно**, потому что "Забито" и "Пропущено" были получены из `_parse_score` и уже являются `int`.
  - **Ненужность:** **Необходима** для сортировки и отображения.

- **Конвертация 4:** `standings["Ш"] = standings["Забито"].astype(str) + "-" + standings["Пропущено"].astype(str)`
  - **Назначение:** Формирует строковое представление шайб (например, "150-120").
  - **Безопасность:** **Безопасно**. `astype(str)` на целых числах работает корректно.
  - **Ненужность:** **Необходима** для отображения в турнирной таблице в формате "З-П".

- **Конвертация 5:** `standings["Sub_Место"] = np.arange(1, len(standings) + 1)`
  - **Назначение:** Создает последовательность чисел от 1 до N для ранжирования.
  - **Безопасность:** **Безопасно**. `np.arange` создает числовой массив.
  - **Ненужность:** **Необходима** для присвоения мест.

- **Конвертация 6:** `standings["Место"] = standings["Sub_Место"]`
  - **Назначение:** Копирует временное поле места.
  - **Безопасность:** **Безопасно**.
  - **Ненужность:** **Необходима** как финальное поле.

**4. `KHLSeasonAnalyzer.get_conference_tables` метод:**

- **Конвертация 1:** `mapped_standings["Лидер_Дивизиона"] = mapped_standings.index.isin(valid_leaders_idx).astype(int)`
  - **Назначение:** Преобразует булев флаг (является ли команда лидером дивизиона) в `1` или `0`.
  - **Безопасность:** **Полностью безопасно**, как в `_process_match_outcomes`.
  - **Ненужность:** **Не необходима**, но делает код для последующей сортировки более читаемым (`sort_values(by=["Лидер_Дивизиона"] + SORT_COLUMNS, ascending=[False] + SORT_ASCENDING)`). Без этого пришлось бы использовать `mask` или `where` для явного разделения лидеров и остальных.

- **Конвертация 2:** `sub_df["Место"] = np.arange(1, len(sub_df) + 1)`
  - **Назначение:** Присваивает места внутри конференции.
  - **Безопасность:** **Безопасно**.
  - **Ненужность:** **Необходима** для вывода таблицы.

  - **Ненужность:** **Необходима** для реализации функционала записи в файл.

### Выводы и рекомендации

1. **Конвертации в `int`:**
    - Практически все конвертации в `int` **необходимы** для выполнения дальнейших расчетов (суммирование очков, голов, подсчет игр, разницы шайб).
    - Наиболее критичные места (парсинг счета и голов) используют `errors="coerce"` и `fillna(0)`, что делает их **очень устойчивыми к некорректным данным**. Если в CSV файле окажется "не число" там, где ожидалось число, программа не крашнется, а просто примет это значение за 0. Это именно то, что требуется.
    - Конвертация булевых значений (`True`/`False`) в `1`/`0` (`astype(int)`) не является строго необходимой, но делает код более лаконичным и читаемым, что является общим соглашением при работе с Pandas/NumPy.

2. **Возможные "лишние" int:**
    - `astype(int)` после `pd.to_numeric(...).fillna(0)` — это **не лишняя** конвертация. `pd.to_numeric` может вернуть `float` (например, `0.0`), а `astype(int)` приводит его к `0`. Это нормальная практика для приведения к целому типу, когда мы уверены, что число целое.
    - `astype(int)` на булевых значениях. Как уже сказано, не строго необходим, но часто используется для удобства.

3. **Требование "не крашилось"**:
    - Код **хорошо справляется** с этим требованием благодаря использованию `errors="coerce"`, `fillna(0)` и `pd.to_numeric`. В местах, где ожидаются числа, неправильные форматы данных не вызывают сбоев.

4. **Требование "лишних интов"**:
    - Скорее всего, преподаватель имел в виду явные `int()` вызовы, которые могут вызвать `ValueError`, или `astype(int)`, если преобразование может быть небезопасным. В данном коде такие случаи **отсутствуют**. Все `astype(int)` применяются либо к безопасному (уже обработанному) типу данных, либо там, где это оправдано логикой (как с булевыми значениями).

**Итог:**

Код **соответствует требованиям преподавателя**. Конвертации в `int` там, где они есть, либо необходимы для логики, либо применяются к данным, которые уже были безопасно обработаны. Программа должна быть устойчива к некорректным числовым данным в CSV.
